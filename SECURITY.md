# 🔐 Security Policy

Thank you for helping keep this app and its users safe. I take security issues seriously
and appreciate responsible disclosure.

---

## 📱 Supported Versions

I only actively maintain and patch the latest version of the app available on the
**Apple App Store**. Please make sure you're testing against the most current release
before submitting a report.

| Version | Supported |
| ------- | --------- |
| Latest (App Store) | ✅ |
| Any older version  | ❌ |

---

## 🚨 Reporting a Vulnerability

**Please do NOT open a public GitHub issue for security vulnerabilities.**
Public issues are visible to everyone, which could put users at risk before a fix is available.

Instead, please report vulnerabilities using the following private channel:

### GitHub Private Security Advisory
Use GitHub's built-in private reporting tool &#8594; [Report a Vulnerability](../../security/advisories/new)

This creates a private, encrypted thread directly with me and is the fastest way to get a response.

---

## 📋 What to Include in Your Report

The more detail you provide, the faster I can assess and fix the issue. Please try to include:

- **Description** - What is the vulnerability? What component or feature does it affect?
- **Impact** - What could an attacker do if they exploited this? (e.g. access user data, bypass authentication)
- **Steps to reproduce** - A clear, step-by-step way to trigger the issue
- **Proof of concept** - Screenshots, screen recordings, or code snippets if applicable
- **Suggested fix** - If you have one in mind (optional but appreciated!)
- **Your contact details** - So I can follow up with you directly

---

## ⏱️ What to Expect

| Stage | Timeframe |
| ----- | --------- |
| Acknowledgement of your report | Within **48 hours** |
| Initial assessment & severity triage | Within **5 business days** |
| Status update (fix in progress / not applicable) | Within **14 days** |
| Patch release (for confirmed vulnerabilities) | As soon as possible - target **30 days** for critical issues |

I'll keep you updated throughout the process via your chosen reporting channel.

---

## ✅ My Commitment to You

- I will **acknowledge** every legitimate report promptly
- I will **not take legal action** against researchers who report in good faith
- I will **credit you** in release notes if you'd like recognition (just let me know!)
- I will **notify you** when the vulnerability has been patched

---

## 🤝 Responsible Disclosure

I kindly ask that you:

- Give me a reasonable amount of time to fix the issue **before** disclosing it publicly
- Avoid accessing, modifying, or deleting user data during testing
- Only test against your **own account** - do not test against other users' data
- Act in good faith - I'll do the same

I follow a **90-day disclosure window**: if a patch has not been released within 90 days
of a confirmed report, you are free to disclose publicly. I will always try to coordinate
the timing with you.

---

> 🌍 Please write in **English** so I can fully understand and act on your report as quickly as possible.
